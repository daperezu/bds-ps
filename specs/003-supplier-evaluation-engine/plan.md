# Implementation Plan: Supplier Evaluation Engine

**Branch**: `003-supplier-evaluation-engine` | **Date**: 2026-04-16 | **Spec**: [spec.md](spec.md)
**Input**: Feature specification from `specs/003-supplier-evaluation-engine/spec.md`

## Summary

Replace the current lowest-price-only supplier recommendation with a domain-computed composite scoring engine. Each quotation on an item is scored 0-5 based on five equally-weighted boolean factors: CCSS compliance, Hacienda compliance, SICOP compliance, electronic invoicing, and price competitiveness. The existing `Supplier.ComplianceStatus` string field is replaced with three boolean fields. Scores are computed on-the-fly (not persisted), displayed on the review screen with a factor breakdown, and the top scorer is recommended and pre-selected.

## Technical Context

**Language/Version**: C# / .NET 10.0
**Primary Dependencies**: ASP.NET MVC, Entity Framework Core 10.0, ASP.NET Identity, .NET Aspire
**Storage**: SQL Server (Aspire-managed container for dev, dacpac schema management)
**Testing**: Playwright for .NET (NUnit), NUnit (unit tests), NSubstitute (mocking)
**Target Platform**: Linux/Windows server (Aspire-orchestrated)
**Project Type**: Web application (server-side MVC)
**Performance Goals**: Score computation adds negligible overhead (in-memory calculation over 2-5 quotations per item)
**Constraints**: Scoring logic must be unit-testable with zero database dependencies
**Scale/Scope**: Typical item has 2-5 quotations; typical application has 1-10 items
